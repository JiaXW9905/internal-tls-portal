# 产业互联网内部门户 产品需求文档（PRD）

## 1. 文档信息
- 产品名称：产业互联网内部门户（Industrial Internet Internal Portal）
- 当前版本基线：`1.0.0`
- 核心服务：TLS证书管理、[未来服务...]
- 文档目的：定义门户系统的业务目标、服务架构、权限模型与验收标准，作为后续开发、测试、运维与文档编写的单一事实来源（SSOT）。

## 2. 背景与目标

### 2.1 产品定位
产业互联网内部门户是一个统一的企业服务平台，用于集中管理和访问多个内部业务系统。当前首个接入服务为TLS证书管理，后续将陆续接入更多企业服务。

### 2.2 演进历史
- **v0.x**: 单一TLS证书管理系统，提供证书申请、签发、下载与生命周期管理
- **v1.0**: 架构升级为多服务门户，引入RBAC权限模型，TLS证书管理作为首个接入服务

### 2.1 业务目标
- 让申请、签发、下载流程可追踪、可审计、可维护。
- 通过角色与数据权限控制，确保“最小权限访问”。
- 支持稳定升级与快速回滚，降低发布风险。

### 2.2 非目标
- 不覆盖 CA 实际签发流程自动化（系统只管理申请与证书包文件）。
- 不对接外部 SSO（当前沿用邮箱注册/登录体系）。
- 不支持多租户隔离（当前为单实例内部系统）。

## 3. 角色与权限
- `admin`：平台管理员（用户/设置管理、签发、总览查看）。
- `dev`：研发签发角色（签发、撤销、总览查看）。
- `service`：申请人角色（提交申请、查看/下载自己的申请）。
- `product`：产品/查看角色（总览查看）。

### 3.1 数据可见性规则（关键）
- 在“证书申请”和“证书签发”相关数据接口中，`admin/dev` 默认仅可见“与自己相关”的记录：
  - 自己提交的申请，或
  - 自己签发的证书记录。
- `service` 仅可见自己提交的申请。
- `product` 不进入签发页，仅可访问总览视图。

## 4. 核心流程
1. 用户注册（公司邮箱）并登录。
2. 申请人提交证书申请（客户名/SKU/VID/类型）。
3. 研发/管理员在签发页上传 ZIP 并完成签发。
4. 申请人在“我的申请”下载 ZIP（若文件缺失则按钮不可操作）。
5. 签发后 7 天内允许撤销；超过 7 天不可撤销。
6. 总览页按状态/关键字/时间条件查看并导出。

## 5. 功能需求

### 5.1 认证与账号
- 注册、登录、登出、忘记密码、修改密码。
- 邮箱验证码流程（注册与重置密码）。
- 管理员可管理用户角色、重置用户密码、编辑用户信息。

### 5.2 申请管理
- 创建申请字段：
  - `customer_name`
  - `product_sku`
  - `vid`
  - `requester_name`
  - `request_type`（`new`/`delete`/`renew`）
- 仅 `pending` 状态允许“工单撤回”。

### 5.3 签发管理
- 仅 `admin/dev` 可执行签发。
- 上传 ZIP 文件和解压密码。
- 签发成功后状态变更为 `issued`。

### 5.4 证书撤销规则
- 仅存在活跃证书时可撤销。
- 仅 `admin/dev` 可执行撤销。
- 签发超过 7 天（按证书创建时间）禁止撤销。
- 前端超期记录操作列显示 `-`（不显示“可撤销”动作）。

### 5.5 证书总览
- 支持筛选：状态、关键字、时间、即将过期（60 天）。
- 支持导出 CSV。
- 过期时间展示规则：
  - 有明确过期时间字段时，优先显示该值；
  - 无过期时间时，按 `证书创建时间 + 730 天（365*2）` 计算。

### 5.6 文件可用性
- 若记录存在但证书文件不存在：
  - “我的申请”中下载按钮置为不可操作；
  - 总览和签发页应明确提示文件缺失或不可下载。

### 5.7 版本可见性
- 所有页面右下角显示版本角标（来源 `public/version.js`）。

## 6. 非功能需求
- 可用性：核心页面可访问，关键 API 稳定返回。
- 安全性：会话鉴权、角色鉴权、最小数据可见范围。
- 可运维性：支持 Docker 化部署、升级、回滚。
- 可追溯性：`CHANGELOG` 持续维护版本变更。

## 7. 数据模型（摘要）
- `users`：用户、角色、邮箱验证状态。
- `requests`：申请主表、状态、申请类型、签发时间。
- `request_certificates`：证书文件路径、密码、签发人、撤销信息。
- `system_settings`：邮件与系统配置。

## 8. 验收标准（UAT）
- 角色登录后仅看到符合权限的数据范围。
- 证书总览可正确加载 >200 条数据，并分页展示。
- 无过期时间证书可正确展示“签发时间 + 730 天”。
- 超过 7 天证书不能撤销，且界面不展示可操作按钮。
- 无文件证书在“我的申请”中下载按钮不可点击。
- 页面右下角稳定显示当前版本号。

## 9. 风险与约束
- SQLite 单机部署，横向扩展能力有限。
- 历史数据质量（缺失文件、缺失字段）会影响展示一致性。
- 升级脚本与发布流程需严格遵循备份与回滚规范。

## 10. 门户与RBAC功能（v1.0新增）

### 10.1 门户首页
- 服务卡片式导航，展示用户可访问的所有服务
- 欢迎信息和用户个性化展示
- 响应式设计，支持桌面和移动端

### 10.2 RBAC权限系统
- **数据库表**：8张RBAC相关表（服务、角色、权限、映射、审计）
- **权限检查**：中间件级别的细粒度权限控制
- **审计日志**：完整的服务访问日志和权限变更历史
- **向后兼容**：自动迁移现有用户角色到RBAC体系

### 10.3 服务接入机制
新服务接入步骤：
1. 在 `portal_services` 表注册服务
2. 定义服务的权限列表（permissions表）
3. 定义服务的角色（roles表）
4. 关联角色与权限（role_permissions表）
5. 为用户分配角色（user_roles表）
6. 在 `sidebar-nav.js` 中添加导航配置

### 10.4 API命名空间
- `/api/portal/*` - 门户管理接口（服务列表、权限查询、角色管理）
- `/api/tls/*` - TLS服务接口（预留，当前保持 `/api/*` 兼容）
- 未来服务：`/api/{service-id}/*`
